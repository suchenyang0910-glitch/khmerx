# 风控平台管理后台｜页面设计（桌面端优先）

## 全局设计规范
- Layout：桌面端采用「顶部导航 + 左侧侧边栏 + 右侧内容区」的混合布局；内容区使用 CSS Grid（表格/卡片区）+ Flex（表单/工具条）。
- Breakpoints：
  - ≥1280：侧边栏常驻（240px），内容区 1200px 以内居中（可配置全宽）。
  - 768–1279：侧边栏可折叠（图标模式 64px）。
  - <768：仅保留核心读写（本版本以桌面优先，不强行完善移动端）。
- Global Tokens：
  - 背景：#0B1220（深色）/ #F7F8FA（浅色二选一，默认浅色）
  - 主色：#2563EB；成功 #16A34A；警告 #F59E0B；危险 #DC2626
  - 字体：14px 基础；标题 18/20/24；等宽用于 ID 与 JSON 摘要
  - 按钮：Primary/Secondary/Danger；hover 提升亮度 6–8%；disabled 降低 40% 不透明度
  - 表格：斑马纹 + 粘性表头；支持列宽拖拽（可选）
- 交互通用：
  - 全局顶部放置用户菜单（角色、退出）、环境标识（dev/staging/prod）
  - 所有写操作需要二次确认（发布/回滚/下线/权限变更/导出）
  - Toast：成功/失败；错误信息包含可执行指引（缺权限/字段缺失/网络异常）

### Meta Information
- Title：控制台 - 风控平台管理后台
- Description：指标概览、告警与快捷操作

## 3) 风控规则管理 /rules
### Meta Information
- Title：风控规则管理 - 风控平台管理后台
- Description：规则列表、编辑、版本与发布

### Page Structure
- 左：规则列表（表格）
- 右：详情/编辑抽屉（Drawer）或同页 Tabs（“配置”“版本”“发布记录”）

### Sections & Components
- 顶部工具条
  - 搜索（名称/ID）、筛选（状态/标签/创建人）、新建规则按钮
- 规则表格
  - 列：名称、状态、优先级、标签、当前版本、更新时间、操作（编辑/发布/下线）
  - 行内状态徽章（draft/enabled/disabled）
- 规则编辑（Form）
  - 基础信息：名称、描述、标签、优先级、开关
  - 条件与动作：JSON 编辑器（受控）+ 可选表单化子组件（最小版仅 JSON）
  - 保存草稿、生成版本按钮
- 校验与测试（Panel）
  - 样例输入（JSON）→ 运行校验 → 展示命中结果/错误定位
- 版本与发布（Tabs）
  - 版本列表：版本号、变更说明、创建人、时间
  - 发布弹窗：发布类型（灰度/全量/回滚/下线）、说明、二次确认

权限控制（前端必须显式隐藏/禁用）：
- 无 rules.write：隐藏新建/保存/生成版本
- 无 rules.publish：禁用发布/回滚/下线

## 4) 命中与处置中心 /cases
### Meta Information
- Title：命中与处置中心 - 风控平台管理后台
- Description：命中事件检索、详情与处置

### Page Structure
- 上：筛选区（2 行表单）
- 中：命中列表表格（可多选）
- 右：事件详情抽屉（含处置组件）

### Sections & Components
- 筛选区
  - 时间范围、规则、对象ID、风险等级、处置状态
  - 查询/重置、导出 CSV（需 export.csv 权限）
- 列表表格
  - 列：时间、对象ID、规则、风险等级、分数、状态、最近处置人
- 事件详情 Drawer
  - 命中摘要：命中规则链、关键字段摘要
  - 输入快照：JSON 折叠展示（脱敏提示）
  - 处置历史时间线
- 处置动作
  - 动作选择（放行/拦截/人工复核/黑白名单）+ 备注
  - 提交后刷新列表状态，并写入审计

权限控制：
- 无 cases.dispose：详情页只读，隐藏处置按钮

## 5) 系统管理（权限 + 日志与审计）/system
### Meta Information
- Title：系统管理 - 风控平台管理后台
- Description：用户/角色/权限与审计日志

### Page Structure
- 页面 Tabs：用户管理｜角色与权限｜审计日志｜登录与访问日志

### Sections & Components
- 用户管理 Tab
  - 用户表格：邮箱、姓名、状态、角色、最近登录
  - 邀请用户弹窗：邮箱、显示名、初始角色
  - 停用/启用、重置角色（均需二次确认）
- 角色与权限 Tab
  - 左：角色列表；右：权限矩阵（checkbox 表格）
  - 保存变更后写入审计（记录 diff）
- 审计日志 Tab
  - 过滤：操作者、动作、对象类型/ID、时间
  - 列表：时间、操作者、动作、对象、IP
  - 详情：diff_json 只读查看；导出
- 登录与访问日志 Tab
  - 登录成功/失败、IP、UA、时间；按用户与时间过滤

权限控制：
- 仅 system.admin 可修改用户/角色/权限；audit.read 才可查看审计详情与导出
