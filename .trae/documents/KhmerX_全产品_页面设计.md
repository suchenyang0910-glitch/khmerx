## 2. 全局交互规则

必须规则
- 关键操作二次确认：发布借款 / 确认出借 / 确认收款 / 确认还款 / 发起争议
- 金额四件套同屏：借款金额 / 平台费或利息 / 实际到账 / 到期应还
- 所有状态必须有“下一步提示”：当前在第几步 + 下一步做什么
- 风控限制必须解释原因：不可只写“不可操作”

身份与鉴权
- 每次请求带：`Authorization: tma <telegram_init_data>`
- 用户唯一识别：`tg_id`（不认 bot_id）

### 4.4 借款 Borrow
页面目标
- 10 秒创建借款单；0 纠纷

控件
1) 额度卡
- 可借额度、信用等级

2) 金额输入
- 步进器：`- / +`（每次 10）
- 输入框
- 滑块：50 → max

3) 期限选择
- 7/14/30（按钮组）
- 新用户默认只显示 7 天（若后端有标记）

4) 费用计算卡（必须）
- 借款金额
- 平台/利息
- 实际到账（重点）
- 到期应还（重点）

5) 风险提示卡
- 按时还款可提升额度
- 逾期会降低信用分

6) 提交按钮
- 确认发布借款

状态
- profile_incomplete：按钮显示「先绑定资料」并跳 ProfileSetup
- calculating：调用 `POST /p2p/calculate`
- ready：按钮可点
- submitting：发布中

点击动作
- 点击发布 → 二次确认弹窗（展示四件套）
- 确认后 → `POST /offers`
- 成功：Toast「借款已发布」→ 跳转 Trades

异常提示
- 超过额度：自动回到最大额度 + 提示「最高可借 $X」
- `ABA_REQUIRED`：提示「请先绑定 ABA」
- `RISK_REJECTED`：提示后端返回的原因

风控解释（必须出现的原因模块）
- 标题：为什么可能无法借款/额度变化
- 列表（最多 3 条）：reasons
- 行动：actions（跳转到资料完善/查看信用详情）

### 4.8 交易详情 TradeDetail（核心）
页面目标
- 用户永远知道：我在哪一步、我要做什么、还有多久

控件
1) 状态进度条
- 匹配 → 打款 → 确认 → 还款 → 完成

2) 当前状态卡（必须）
- 当前状态：等待打款 / 已上传凭证 / 还款中 / 已逾期 / 争议中
- 下一步：明确动作
- 倒计时：matched 状态 24h

倒计时 UI 规则（必须）
- 显示格式：`HH:MM:SS`
- 低于 1h：变为橙色
- 到 0：弹提示并刷新状态

下一步提示文案（必须）
- 每个状态必须显示“当前/下一步”，例如：
  - 当前：等待打款
  - 下一步：放款人需在 24 小时内转账并上传凭证

3) ABA 信息
- 账号 + 姓名

4) 操作区（动态按钮）
- matched：放款人上传打款凭证；借款人等待
- lend_confirmed：借款人确认收款/发起争议；放款人等待
- repaying：借款人上传还款；放款人确认收到还款
- overdue：立即还款/联系客服
- dispute：补充证据/查看进度

5) 凭证上传控件
- 上传截图
- 金额输入（必填）
- 备注输入
- 提交

6) 还款计划
- 第几期、到期时间、应还金额、状态
- 明天到期提示：黄色
- 逾期提示：红色

7) 争议区
- 发起争议
- 补充证据
- 查看争议状态

异常提示
- `INVALID_STATUS`：提示「当前状态不可操作」
- `DISPUTE_LOCKED`：提示「交易争议中，暂不可继续操作」

API

### 4.15 争议仲裁 Dispute（发起/补充证据/查看进度）

- 上传证据

交互

API
- `POST /api/v1/disputes`
- `GET /api/v1/disputes/{id}`

## 9. 最终闭环判断（V1）

- 用户能借款
- 出借人能接单
- 双方能上传和确认凭证
- 逾期/争议有处理入口
- 信用和风控能反馈到额度
