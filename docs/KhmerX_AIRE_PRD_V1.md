## 四、核心商业模式（最终版）

### （一）SaaS模式
**标准化云端风控平台：**
- 企业注册即用
- 按年/月订阅
- 分版本收费

**适用场景：**手机租赁、小额分期、租房、本地商户。

### （二）RaaS模式（核心壁垒）
**Risk-as-a-Service**
按效果收费，按“风控结果价值”收费：
- 拦截欺诈订单
- 降低坏账率
- 降低逾期率
- 降低拒付率

### （三）API开放平台模式
提供以下API，按调用收费：
- 信用查询API
- 风险评级API
- 反欺诈API
- 黑名单查询API
- 设备风险API

### （四）私有化部署模式
面向：银行、金融机构、大型商户、供应链企业。
支持：
- 本地化部署
- 独立数据库
- 独立模型
- 独立规则库

**技术实现：**
- 支持：IF/ELSE、权重评分、风险标签、动态阈值、多场景规则切换。

**可视化规则拖拽支持：**
风险节点拖拽、风险树配置、审批流配置、风险等级联动、动态规则学习。

**系统自动：**
根据违约数据优化规则、调整阈值、调整权重、自动生成建议规则。

**评分维度：**

## 十四、AI模型中心设计（MLOps架构）

### （一）模型分层
1. **信用评分模型**
   - **用途**：信用等级、授信额度、风险预测。
   - **技术**：XGBoost、LightGBM。
2. **欺诈识别模型**
   - **用途**：羊毛党、团伙欺诈、虚假身份。
   - **技术**：GNN图神经网络、Isolation Forest。
3. **行为异常模型**
   - **用途**：高频申请、异常设备、IP漂移。
   - **技术**：时序模型、聚类模型。
4. **LLM风控解释模型**
   - **用途**：自动生成风控报告、AI审核解释、风险建议。
   - **技术**：Qwen、DeepSeek、Llama。

### （二）MLOps模型生命周期
**模型训练流程：**
数据采集 → 数据清洗 → 特征工程 → 模型训练 → 模型评估 → 灰度测试 → 上线部署 → 效果监控 → 自动迭代。

### （三）模型版本管理
支持：模型灰度发布、A/B测试、回滚、自动监控、漂移检测。

## 十八、KhmerX最核心战略（非常重要）
**第一阶段不要做“大而全”。必须聚焦：**

1. **手机租赁风控**
   - 原因：数据最容易闭环、最容易形成黑名单、最容易形成联邦风控、最容易产生现金流。
