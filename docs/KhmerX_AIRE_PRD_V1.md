# KhmerX AI智能信用风控评估平台
## SaaS + RaaS + API开放平台 金融级深度技术开发需求方案（V1.0 Final）

---

## 一、项目战略定位（顶层定义）

**项目名称**
- **英文名称**：KhmerX AI Risk Engine（KX-AIRE）
- **中文名称**：KhmerX AI智能信用风控决策引擎

## 二、项目核心定位

打造面向：
- 东南亚市场
- 跨境商贸
- 手机租赁
- 小微信贷
- 分期消费
- 租房租赁
- 供应链账期

的 **金融级 AI 风险决策基础设施平台**。

平台采用四位一体商业模式：
1. SaaS订阅
2. RaaS结果付费
3. API开放平台
4. 私有化部署

## 三、项目终极目标

构建：**东南亚版「风控基础设施网络」**

实现：
- AI信用评分
- AI反欺诈
- AI风险量化
- AI自动审批
- AI异常识别
- AI违约预测
- AI风控报告
- 行业黑名单联盟
- 风险共享网络

最终形成：**KhmerX 联邦风控生态系统**

## 四、核心商业模式（最终版）

### （一）SaaS模式
**标准化云端风控平台：**
- 企业注册即用
- 按年/月订阅
- 分版本收费

**适用场景：**手机租赁、小额分期、租房、本地商户。

### （二）RaaS模式（核心壁垒）
**Risk-as-a-Service**
按效果收费，按“风控结果价值”收费：
- 拦截欺诈订单
- 降低坏账率
- 降低逾期率
- 降低拒付率

### （三）API开放平台模式
提供以下API，按调用收费：
- 信用查询API
- 风险评级API
- 反欺诈API
- 黑名单查询API
- 设备风险API

### （四）私有化部署模式
面向：银行、金融机构、大型商户、供应链企业。
支持：
- 本地化部署
- 独立数据库
- 独立模型
- 独立规则库

## 五、系统整体技术架构（金融级）

### （一）系统总体架构
```text
用户/商户系统
        ↓
API Gateway（统一网关）
        ↓
权限认证中心
        ↓
风控规则引擎
        ↓
AI评分模型中心
        ↓

        ↓

        ↓

计费中心 / SaaS / RaaS
```

## 七、金融级核心模块（完整版）

### （一）权限与身份安全中心
**多角色权限体系：**
超级管理员、商户管理员、风控审核员、API调用员、运维管理员、财务管理员。

**安全体系支持：**
MFA双重认证、Google Authenticator、人脸识别、设备绑定、IP白名单、异地登录预警、行为风控。

### （二）AI风控规则引擎（核心壁垒）
**核心定位：**构建可视化动态风控规则系统，无需工程师改代码。

**技术实现：**
- 支持：IF/ELSE、权重评分、风险标签、动态阈值、多场景规则切换。
- 建议技术：Drools、Aviator、QLExpress。

**可视化规则拖拽支持：**
风险节点拖拽、风险树配置、审批流配置、风险等级联动、动态规则学习。

**系统自动：**
根据违约数据优化规则、调整阈值、调整权重、自动生成建议规则。

### （三）AI信用评分中心
**评分维度：**
| 维度 | 示例 |
| :--- | :--- |
| 身份信用 | KYC |
| 履约历史 | 逾期率 |
| 设备风险 | 多设备 |
| 行为风险 | 高频申请 |
| 订单风险 | 异常订单 |
| 财务能力 | 收入 |
| 社交风险 | 关联团伙 |
| 行业风险 | 黑名单 |

**输出结果生成：**
100分信用评分、AAA-D等级、授信额度、风险建议、违约概率。

### （四）AI反欺诈中心
**核心能力：**
- **身份欺诈**：虚假身份、假证件、虚拟号码、人脸不一致。
- **团伙欺诈**：多头借贷、羊毛党、团伙关联、设备共享。
- **行为异常**：高频申请、异常时间、IP漂移、GPS异常、设备指纹。

**支持设备指纹技术：**
浏览器指纹、手机设备ID、Canvas指纹、WebRTC指纹。

### （五）AI特征工程中心（核心）
**Feature Store 统一管理：**风险特征、行为特征、信用特征、设备特征。

**技术架构：**
| 数据类型 | 存储方案 |
| :--- | :--- |
| 实时特征 | Redis |
| 历史特征 | ClickHouse |
| 非结构化 | MongoDB |

### （六）联邦黑名单中心（超级壁垒）
**平台级黑名单网络支持：**
商户共享黑名单、行业共享黑名单、风险同步预警、风险标签体系（欠租、跑路、欺诈、恶意拒付、高风险设备、团伙关联等）。

### （七）事件驱动系统（高扩展）
**Event Bus Architecture**，采用 Kafka。

**核心事件：**
| 事件 | 描述 |
| :--- | :--- |
| `user_apply` | 用户申请 |
| `fraud_detected` | 欺诈命中 |
| `blacklist_hit` | 黑名单命中 |
| `risk_rejected` | 风险拒绝 |
| `billing_created` | 账单生成 |

### （八）RaaS结果计费中心
- **自动量化**：系统自动计算拦截率、坏账下降率、风险降低值、逾期降低值。
- **自动结算**：支持基础费、效果提成、自动分佣、自动账单。

### （九）API开放平台
| API | 功能 |
| :--- | :--- |
| `/risk/check` | 风控查询 |
| `/credit/score` | 信用评分 |
| `/fraud/check` | 欺诈检测 |
| `/blacklist/check` | 黑名单查询 |
| `/device/risk` | 设备风险 |

**API能力支持：**OpenAPI、SDK、API Key、OAuth2、调用统计、调用限流。

### （十）数据安全与合规中心
- **合规能力**：符合中国个人信息保护法、数据安全法、GDPR、东南亚本地数据法规。
- **数据安全支持**：AES256加密、数据脱敏、数据审计、权限隔离、数据水印、操作追踪。

### （十一）Explainable AI（可解释AI）
**金融级关键模块支持：**
风险原因解释、决策链回放、风险来源分析、AI评分解释。避免“AI黑盒”。

### （十二）报表与BI系统
- **实时看板**：展示风险趋势、坏账趋势、黑名单增长、拦截率、API调用量。
- **自动报告生成**：月报、季报、年报、风险复盘。
